## Template Usage Notes

**适用场景**:
- 新产品/新功能的需求定义
- 版本规划 (如 v2.0.0 MVP 范围)
- 跨团队沟通的需求基线

**PRD → User Story → OpenSpec 关系**:
```
PRD (What & Why)     →  User Stories (用户价值)  →  OpenSpec (How)
宏观需求描述              细粒度需求点                技术实现方案
```

**小团队简化建议**:
- 可省略 RACI 职责分配
- 功能清单使用 `✅/❌` 标记即可
- 成功标准保持 3-5 条
