# Conventions Summary

> **Sources**: `standards/conventions/*.md`

## Git Commit

**Format**: `<type>(<scope>): 中文 / English`

| Type | Usage |
|------|-------|
| feat | New feature |
| fix | Bug fix |
| docs | Documentation |
| refactor | Refactoring |
| test | Testing |
| chore | Misc |

**Project Rules**:
1. NO AI signatures ("Generated by Claude Code")
2. NO vague messages ("update", "fix bug", "WIP")
3. Bilingual format required

## Naming Conventions

- **Files**: snake_case (Python, Dart), kebab-case (URLs)
- **Classes**: PascalCase
- **Variables**: snake_case (Python), camelCase (Dart)
- **Constants**: SCREAMING_SNAKE_CASE
- **Docs**: `{domain}-{type}-{topic}.md` (recommended, e.g., `be-dev-api.md`)

## Content Integrity

- No fictional data in formal docs
- Date format: YYYY-MM-DD (ISO 8601)
- Mark examples clearly: "// Example"
- AI-generated content must be reviewed
