# Conventions Summary

> **Version**: 1.1.0
> **Sources**: `standards/conventions/*.md`

## Git Commit

**Format**: `<type>(<scope>): 中文 / English`

| Type | Usage |
|------|-------|
| feat | New feature |
| fix | Bug fix |
| docs | Documentation |
| refactor | Refactoring |
| test | Testing |
| chore | Misc |

**Enhanced Markers** (for complex commits):
```
🤖 Executed-By: {subagent_type} subagent
📋 Context: {Phase}-{Cycle} {context}
🔗 Module: {module_name}
```

**TDD Phase Markers**:
```
TDD Phase: RED    # Write failing test
TDD Phase: GREEN  # Minimal implementation
TDD Phase: REFACTOR  # Optimize structure
```

**Project Rules**:
1. NO AI signatures ("Generated by Claude Code", "Co-Authored-By")
2. NO vague messages ("update", "fix bug", "WIP")
3. Bilingual format required

## Naming Conventions

- **Files**: snake_case (Python, Dart), kebab-case (URLs)
- **Classes**: PascalCase
- **Variables**: snake_case (Python), camelCase (Dart)
- **Constants**: SCREAMING_SNAKE_CASE
- **Docs**: `{domain}-{type}-{topic}.md` (recommended, e.g., `be-dev-api.md`)
- **Branches**: `feature/{module}/{task-id}-{short-desc}`

## Content Integrity

- No fictional data in formal docs
- Date format: YYYY-MM-DD (ISO 8601)
- Mark examples clearly: "// Example"
- AI-generated content must be reviewed
