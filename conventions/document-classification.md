# 文档分类规范

> **Version**: 1.0.0
> **Status**: Active
> **Purpose**: 定义项目文档的分类体系，确保文档组织清晰、维护有序

---

## 1. 概述

本规范定义了项目文档的统一分类体系，确保：
- 文档组织结构清晰
- 维护责任明确
- 便于查找和管理
- 支持自动化工具处理

---

## 2. 文档分类体系

### 2.1 四大分类

| 分类 | 说明 | 特点 | 维护方式 |
|------|------|------|----------|
| **规范性文档** | 定义标准、规范和指南 | 稳定、权威、指导性 | 手动维护，需审核 |
| **被动更新文档** | 系统或工具自动生成 | 自动生成、数据驱动 | 自动更新，不手动修改 |
| **主动维护文档** | 需人工定期更新 | 反映当前状态 | 指定责任人定期更新 |
| **数据记录** | 结构化数据存储 | JSON格式、可引用 | 脚本自动更新 |

---

## 3. 规范性文档

### 3.1 定义

规范性文档定义项目的标准、规范和指导原则，是项目开发的权威参考。

### 3.2 特点

```yaml
特点:
  - 稳定性高，变更频率低
  - 需要团队讨论和审核后才能修改
  - 作为其他文档的参考基准
  - 指导开发行为和决策

维护方式:
  - 版本控制
  - 变更需审核
  - 定期评审
```

### 3.3 子类别

#### 核心规范文档

```yaml
定义: 定义项目核心标准和规范
示例:
  - 命名规范
  - 代码风格规范
  - Git提交规范
  - 文档编写规范
特点: 变更频率最低，影响范围最广
```

#### 组件/模板文档

```yaml
定义: 定义可重用的文档组件和模板
示例:
  - 文档头部模板
  - API文档模板
  - 需求文档模板
特点: 需要保持一致性，便于复用
```

### 3.4 目录建议

```
docs/
├── standards/           # 核心规范文档
│   ├── naming-*.md
│   ├── style-*.md
│   └── workflow-*.md
└── templates/           # 模板文档
    ├── api-template.md
    └── requirement-template.md
```

## 4. 被动更新文档

### 4.1 定义

被动更新文档由系统或脚本自动生成，不需要手动维护。

### 4.2 特点

```yaml
特点:
  - 自动生成，数据驱动
  - 反映系统实时状态
  - 不应手动修改
  - 格式统一，便于解析

维护方式:
  - 脚本自动更新
  - 定时任务触发
  - CI/CD集成
```

### 4.3 子类别

#### 报告类文档

```yaml
定义: 详细的数据分析报告
示例:
  - 文档健康度报告
  - 测试覆盖率报告
  - 依赖关系报告
  - 代码质量报告
特点: 数据详细，按功能分类
```

#### 摘要类文档

```yaml
定义: 简明的状态概述
示例:
  - 项目状态摘要
  - 模块进度摘要
  - 每日/每周摘要
特点: 信息精简，便于快速了解
```

#### 仪表板类文档

```yaml
定义: 可视化状态展示
示例:
  - 项目仪表板
  - 性能监控仪表板
  - 质量指标仪表板
特点: 图表为主，指标突出
```

#### 数据类文档

```yaml
定义: 原始数据和元数据
示例:
  - 文档索引数据
  - 统计数据
  - 配置数据
特点: 结构化存储，用于生成其他文档
```

### 4.4 目录建议

```
docs/
└── generated/           # 自动生成文档
    ├── reports/         # 报告类
    │   ├── health/
    │   ├── coverage/
    │   └── quality/
    ├── summaries/       # 摘要类
    ├── dashboards/      # 仪表板类
    └── data/            # 数据类
```

## 5. 主动维护文档

### 5.1 定义

主动维护文档需要团队成员定期更新，反映项目当前状态。

### 5.2 特点

```yaml
特点:
  - 需要定期更新
  - 反映当前实际状态
  - 有明确的责任人
  - 更新频率较高

维护方式:
  - 指定责任人
  - 定期更新周期
  - 变更记录
```

### 5.3 常见类型

```yaml
项目规划文档:
  - 开发计划
  - 里程碑规划
  - 资源分配

架构设计文档:
  - 系统架构
  - 技术选型
  - 接口设计

实施状态文档:
  - 实施进度
  - 问题跟踪
  - 决策记录

API文档:
  - API参考
  - 集成指南
  - 变更日志
```

### 5.4 目录建议

```
docs/
└── maintained/          # 主动维护文档
    ├── architecture/    # 架构设计
    ├── planning/        # 项目规划
    ├── development/     # 开发文档
    │   ├── backend/
    │   ├── frontend/
    │   └── mobile/
    └── api/             # API文档
```

## 6. 数据记录

### 6.1 定义

数据记录是存储在文件中的结构化数据，用于在多个文档中引用。

### 6.2 特点

```yaml
特点:
  - 结构化数据（JSON/YAML）
  - 系统自动更新
  - 可在多处引用
  - 单一数据源

维护方式:
  - 通过脚本更新
  - 不手动编辑
  - 版本控制
```

### 6.3 常见类型

```yaml
文档元数据:
  - 文档索引
  - 分类映射
  - 引用关系

项目数据:
  - 进度数据
  - 统计数据
  - 配置数据

历史记录:
  - 变更历史
  - 版本记录
  - 审计日志
```

### 6.4 目录建议

```
docs/
└── database/            # 数据记录
    ├── document-index.json
    ├── classification-map.json
    └── statistics.json
```

## 7. 文档分类标识

### 7.1 YAML Frontmatter

每个文档应在开头使用 YAML frontmatter 标明分类：

```yaml
---
classification: 规范性文档
subcategory: 核心规范文档
status: active
created_at: '2025-12-14'
updated_at: '2025-12-14'
version: 1.0.0
---
```

### 7.2 分类值定义

```yaml
classification:
  - 规范性文档
  - 被动更新文档
  - 主动维护文档
  - 数据记录

subcategory:
  规范性文档:
    - 核心规范文档
    - 组件文档
    - 模板文档

  被动更新文档:
    - 报告类文档
    - 摘要类文档
    - 仪表板类文档
    - 数据类文档

  主动维护文档:
    - 架构文档
    - 规划文档
    - 开发文档
    - API文档

  数据记录:
    - 索引数据
    - 统计数据
    - 配置数据

status:
  - active      # 当前有效
  - draft       # 草稿状态
  - deprecated  # 已废弃
  - archived    # 已归档
```

## 8. 分类迁移

### 8.1 迁移流程

```yaml
Step 1 - 评估:
  - 确定文档当前分类
  - 确定目标分类
  - 评估影响范围

Step 2 - 更新:
  - 更新文档 frontmatter
  - 移动到对应目录
  - 更新引用链接

Step 3 - 验证:
  - 检查链接有效性
  - 验证分类正确性
  - 更新索引
```

### 8.2 迁移检查清单

```yaml
迁移前:
  - [ ] 备份原文档
  - [ ] 识别所有引用
  - [ ] 确认目标位置

迁移中:
  - [ ] 更新 classification 字段
  - [ ] 移动文件
  - [ ] 更新所有引用链接

迁移后:
  - [ ] 验证链接有效
  - [ ] 更新文档索引
  - [ ] 通知相关人员
```

## 10. 维护责任矩阵

| 分类 | 创建者 | 维护者 | 审核者 | 更新频率 |
|------|--------|--------|--------|----------|
| 规范性文档 | Tech Lead | Knowledge Manager | Team | 低 (季度) |
| 被动更新文档 | 系统 | 系统 | - | 自动 |
| 主动维护文档 | 开发者 | 指定责任人 | Tech Lead | 中 (周/月) |
| 数据记录 | 系统 | 系统 | - | 自动 |

## 11. 快速参考

### 11.1 分类判断流程

```
文档是否由系统自动生成？
  ├─ 是 → 被动更新文档
  │       ├─ 数据为主 → 数据记录
  │       └─ 报告/摘要 → 报告/摘要类
  │
  └─ 否 → 需要人工维护
          ├─ 定义标准/规范 → 规范性文档
          └─ 反映当前状态 → 主动维护文档
```

### 11.2 分类标识速查

| 分类 | frontmatter |
|------|-------------|
| 核心规范 | `classification: 规范性文档` |
| 自动报告 | `classification: 被动更新文档` |
| 架构设计 | `classification: 主动维护文档` |
| 索引数据 | `classification: 数据记录` |
